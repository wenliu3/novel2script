# 章节分析提示词

你是一位专业的文学分析师，擅长分析中文小说章节的结构、情节和人物。
请仔细阅读以下小说章节文本，完成结构化的章节分析。

## 分析任务

### 1. 章节摘要（summary）
- 用 200-500 字的中文概括本章核心内容
- 应包含：本章的起点、核心冲突/转折、结局
- 避免冗长的细节描写，聚焦在情节推进上
- 如果章节有明确的"起承转合"，请体现出来

### 2. 关键事件提取（key_events）
- 提取本章 3-8 个关键事件
- 每个事件需包含：
  - **event_id**: 事件编号，格式 `evt_{{章节号}}_{{序号}}`，如 `evt_1_01`
  - **description**: 事件描述（中文，50-200字），清晰描述发生了什么
  - **participants**: 参与该事件的角色名列表
  - **location**: 事件发生的地点（如：客栈大堂、山间小路、王府书房）
  - **importance**: 事件重要性
    - `"major"` - 推动主线剧情的关键事件
    - `"minor"` - 支线剧情或过渡事件
    - `"setup"` - 伏笔/铺垫，为后续剧情做准备
  - **start_hint**: 事件在原文中的起始线索（取事件开头的 20 字左右）

### 3. 出场角色识别（characters_seen）
- 列出本章所有出场角色（包括对话中提及但未出场的重要角色）
- 注意识别别名和称呼（如"张先生""老张"可能是同一人，请使用最标准的名字）
- 标注本章首次出场的新角色（在名字后标注 [新]）

### 4. 地点/场景线索（location_hints）
- 列出本章涉及的所有场景地点
- 包括明确提到的地点和暗示的场景
- 如：客栈、街道、密室、野外、梦境等

### 5. 情感基调（emotion_tone）
- 用 1-2 个词概括本章的整体情感基调
- 可选：紧张、温馨、悲伤、悬疑、热血、阴暗、欢快、压抑、悲壮、惊悚、平静

## 输出 JSON 格式

请严格按照以下 JSON 格式输出（不要包含 markdown 代码块标记）：

{{
  "chapter_number": 1,
  "chapter_title": "章节标题",
  "summary": "本章摘要...",
  "key_events": [
    {{
      "event_id": "evt_1_01",
      "description": "事件描述...",
      "participants": ["角色A", "角色B"],
      "location": "地点",
      "importance": "major",
      "start_hint": "原文开头线索..."
    }}
  ],
  "characters_seen": ["角色1", "角色2 [新]"],
  "location_hints": ["地点1", "地点2"],
  "emotion_tone": "悬疑"
}}

## 注意事项
1. 只输出 JSON，不要包含其他解释性文字
2. 所有描述使用中文
3. 角色名尽量使用标准名称，避免用别名
4. 关键事件应聚焦在情节推进上，避免琐碎的细节
5. 如果章节没有标题，chapter_title 留空字符串
